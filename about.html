About UPick

What is UPick?

UPick is a simple web app that helps teams make decisions together. You can run vote-based decisions with participant limits and completion tracking, or use random selection for quick one-off choices.


Every voice matters

Features

  • Create decisions in Vote or Random mode
  • Track participant progress and vote totals after each submission
  • Send completed vote decisions to the Completed section
  • Keep random decisions available for repeated picks
  • Use decision links to revisit voting rooms

How to Use

  1. Go to the Home
  2. Create a Vote or Random decision
  3. Invite participants to join vote decisions from Home
  4. Review updated progress from the Active Decisions section
  5. Use Complete Decision to archive finished vote decisions in Completed

Technology Stack

UPick is built entirely with custom code — no CMS is used. The languages and frameworks are:

  • HTML5 & CSS3 — structure and styling on all pages
  • JavaScript — API communication
  • Node.js & Express — server-side REST API that handles decisions, participants, and votes
  • MongoDB (via Mongoose) — NoSQL database that persists all decisions and vote data

Hosting & Domain

The frontend static files are served directly by Express, so no separate web host is needed. For deployment the app is suited for platforms like Render because they support Node.js servers and allow a free-tier MongoDB Atlas connection via environment variables.

How the Database Is Used

MongoDB stores every decision that users create, including its title, options, selection mode, participant list, and per-option vote tallies. Because data lives in the database rather than the browser, multiple participants can join the same decision from different devices and all see the same live vote counts. Completed decisions are also persisted so results are never lost when the page is refreshed.

Security

  • Security headers are enabled through Helmet
  • CORS is enabled so the frontend can call the API
  • JSON requests are size-limited to reduce oversized payload risk
  • Duplicate vote checks prevent the same participant from voting twice
